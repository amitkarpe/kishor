# Discussion Framework for Kishor - Workflow

**Status:** Draft v0.2  
**Owner:** Amit  
**Use case:** Reusable source file for creating discussion presentations for Kishor sessions  
**Scope:** Discussion workflow, discussion structure, question flow, and content sequencing. This file does **not** define the visual style system.

---

## 1) Core Instruction

Whenever Amit says:

**Create presentation on <Discussion Topic> using "Discussion Framework for Kishor - Workflow.md"**

follow this workflow:

1. **Do not create the final presentation immediately.**
2. **First create a structured first-draft text outline.**
3. Ask the required clarification questions only if still missing.
4. Treat prior approved defaults as fixed unless Amit changes them.
5. Wait for Amit's approval on the draft text.
6. After approval, create high-level discussion content.
7. Only after that, convert the approved content into a presentation using the selected build tool.

---

## 2) Permanent Tone Rule

Always use a:

- strong
- unapologetic
- civilizational
- proud Hindu

voice for historical and cultural topics.

This means:

- create pride in Hindu identity and civilization
- highlight courage, self-respect, resilience, sacrifice, leadership, and dharmic values
- use confident wording, not weak or diluted wording
- avoid lecture tone; keep it emotionally engaging and discussion-led

But also:

- do **not** use childish insults
- do **not** overcrowd slides
- do **not** turn the presentation into a random history dump

## 4) Clarification Workflow

Before making the first draft, ask only for the runtime fields that are still missing.

### Questions to ask at runtime

1. What is the exact topic?
2. What is the one paragraph about this topic?
3. What is the main goal: pride, knowledge, inspiration, debate, action, or all?
4. Do you already have source points, images, or examples to include?
5. What is the final build tool?

### Important rule

If Amit has already provided any of these in the same thread, do **not** ask again.

---

## 5) Output Workflow

### Step A - First Draft Only

First create a **draft text outline only**.

This draft should include:

- title idea
- 3 to 4 main discussion questions
- suggested emotional direction
- key historical / cultural / practical points
- rough flow of discussion
- likely takeaway and challenge

Do **not** make the final presentation yet.

### Step B - Wait for Approval

Pause and wait for Amit's feedback.

Amit may:

- change tone
- strengthen wording
- remove weak parts
- add examples
- change structure
- reduce or expand sections

### Step C - High-Level Discussion Content

After approval, create high-level content for discussion:

- question sequence
- prompt / hint sequence
- talking points for speaker notes
- emotional landing points
- challenge idea
- closing reflection if needed

### Step D - Convert into Presentation

Only after approval of the text, convert the content into presentation format.

## 6) Default Discussion Presentation Structure

Use this as the default discussion structure unless Amit asks otherwise.

### Presentation flow

1. **Title slide**
2. **Hook Question 1** - one big question only
3. **Prompt / hint slide for Question 1** - 2 to 5 prompts or hints
4. **Discussion slide for Question 1** - question + 3 bullets
5. **Hook Question 2** - one big question only
6. **Prompt / hint slide for Question 2** - 2 to 5 prompts or hints
7. **Discussion slide for Question 2** - question + 3 bullets
8. **Hook Question 3** - one big question only
9. **Prompt / hint slide for Question 3** - 2 to 5 prompts or hints
10. **Discussion slide for Question 3** - question + 3 bullets
11. **Optional Question 4 set** if needed
12. **Reflection slide** if needed
13. **Closing challenge slide**

### Closing rule

- If there is already a reflection slide before closing, the final slide should be **one challenge only**
- If there is **no** reflection slide before closing, the final slide may contain **reflection + challenge**, but the **challenge must be the last point**

---

## 7) Slide Logic Rules

### Hook slide rule

- show **one big question only**
- keep the slide visually clean
- place **2 to 5 prompts or hints in speaker notes**

### Prompt / hint slide rule

- place the 2 to 5 prompts or hints on the next slide
- keep each prompt short
- use them to sustain discussion, not to dump content

### Discussion slide rule

- structure = **question + 3 bullets**
- bullets should support the speaker's spoken discussion
- do not turn the slide into a paragraph summary

### Reflection slide rule

Use when Amit wants:

- personal connection
- moral reflection
- action-oriented emotional landing
- a pause before the final challenge

### Closing challenge rule

- one challenge only by default
- keep it clear, practical, and memorable

---

## 8) Drafting Rules

When drafting content:

- prefer discussion over lecture
- create emotional movement, not flat facts
- keep each question strong and meaningful
- use prompts to unlock discussion
- make the challenge specific enough to act on
- avoid weak generic takeaways

---

## 9) Content Boundaries

Do not:

- overload the deck with dates and facts without purpose
- turn the discussion into a preachy sermon
- weaken the tone into bland textbook language
- use childish insult language
- make the questions vague or lifeless

---

## 10) Quality Checklist Before Final Delivery

Before final delivery, check:

1. Did the process start with draft outline only?
2. Were runtime questions asked only if missing?
3. Are there 3 to 4 strong main discussion questions?
4. Does each question follow hook -> prompt/hint -> discussion?
5. Are prompt slides limited to 2 to 5 short prompts?
6. Do discussion slides use question + 3 bullets?
7. Does the deck land in a strong reflection and/or challenge?
8. Does the final challenge follow the closing rule?

If any answer is "no", revise before delivering.
