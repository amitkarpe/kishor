# Authoring Principles

1. Keep the repo Markdown-first.
2. Keep categories separate.
3. Preserve framework intent.
4. Do not ask unnecessary clarification questions.
5. Prefer stable folder creation over deck mutation.
6. Use high-signal notes instead of long prose.
7. Keep homepage indexing predictable.
8. Preserve provenance for imported materials.
9. Use sample decks as templates, not as rigid final forms.
10. Favor KISS defaults over speculative complexity.
