# Glossary

- **framework**: an authoring workflow document that controls how a certain type of deck should be created
- **style system**: a visual rules document that controls slide appearance and layout behavior
- **legacy deck**: an older presentation or bundle that may need normalization before import
- **normalized deck**: a deck converted into the standard repo shape of `slides.md`, `meta.yml`, `notes.md`, and optional `assets/`
- **slug**: the URL-safe folder name for a presentation
- **Marp**: a Markdown-based slide authoring system
- **Codex**: the coding agent expected to restructure and maintain the repo
- **prompt / hint slide**: a slide used to sustain discussion after a hook question
- **speaker notes**: concise bullet notes that guide live facilitation
