# Assumptions

1. The repo will remain GitHub Pages hosted.
2. Marp Markdown remains the primary presentation format.
3. The repo should scale to many decks over time.
4. The main content categories are `quiz` and `discussion`.
5. New decks should always be added in new folders.
6. Existing decks should not be edited unless explicitly requested.
7. The uploaded framework files are the current authoritative sources.
8. PPTX is secondary and should not drive repo design.
9. Current PPTX bundles are external and intentionally excluded from this ZIP.
10. Legacy import should preserve provenance and normalize structure gradually.
11. Indexing should be simple and predictable.
12. Codex should ask clarifying questions only when a missing detail is truly blocking.
13. Default metadata should stay lightweight.
14. Examples in this package are starter templates, not final site content.
