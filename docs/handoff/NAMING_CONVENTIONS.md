# Naming Conventions

## Package and documentation filenames

Use underscores for package files and helper docs.

Examples:

- `Discussion_Framework_for_Kishor_Workflow.md`
- `Kishor_Presentation_Style_System_Workflow.md`
- `source-manifest.md`

## Presentation folder names

Use lowercase hyphenated slugs.

Format:

- `quiz/<slug>/`
- `discussion/<slug>/`

Examples:

- `discussion/proud-hindu-today/`
- `discussion/what-is-dharma-really/`
- `quiz/indian-science-legends/`

## Slug rules

Slugs should be:

- lowercase
- ASCII-safe
- hyphen-separated
- stable
- short but descriptive

Avoid:

- spaces
- underscores in URLs
- dates in slugs unless needed for versioning
- punctuation beyond hyphens

## Title rules

Use human-readable Title Case for display titles.

Examples:

- `Proud Hindu Today`
- `What Is Dharma, Really?`
- `Indian Science Legends`

## Dates

Use ISO format in metadata:

- `YYYY-MM-DD`

Example:

- `2026-03-15`

## Metadata filenames

Inside each deck folder, keep filenames stable:

- `slides.md`
- `meta.yml`
- `notes.md`

## Asset filenames

Use lowercase hyphenated names.

Examples:

- `hanuman-statue.jpg`
- `vivekananda-portrait.png`
