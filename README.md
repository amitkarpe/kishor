# kishor

Marp-based presentation library for Kishor sessions, published on GitHub Pages.

The repo is now organized as a multi-deck site with two categories:
- `quiz/`
- `discussion/`

Each presentation lives in its own folder and is built into a static site index at:
- `https://amitkarpe.github.io/kishor/`

## Repo layout

- `quiz/<slug>/`: one quiz deck per folder
- `discussion/<slug>/`: one discussion deck per folder
- `docs/sources/`: framework, style, and shared authoring docs imported from the handoff package
- `docs/handoff/`: higher-level operating docs and repo design notes
- `docs/manifests/`: imported manifests from the handoff package
- `legacy-import/`: raw import stubs and references for older decks that still need normalization
- `templates/`: starter files for new quiz and discussion decks
- `scripts/build_site.py`: renders all decks and generates the homepage

## How publishing works

The Pages workflow uses:
- `bash scripts/build-site.sh`

That script:
- finds every deck under `quiz/` and `discussion/`
- validates `meta.yml`
- renders each `slides.md` with Marp into `dist/<category>/<slug>/index.html`
- copies local `assets/` if present
- generates the homepage at `dist/index.html`
- creates `dist/.nojekyll`

## Required per-deck files

Each published deck should normally include:
- `slides.md`
- `meta.yml`
- `notes.md`

Optional:
- `assets/`

## Create a new deck

1. Copy the matching template folder contents from:
   - `templates/quiz/`
   - `templates/discussion/`
2. Create a new folder:
   - `quiz/<slug>/`
   - `discussion/<slug>/`
3. Update `meta.yml`, `slides.md`, and `notes.md`
4. Build locally
5. Push to `main`

## Build locally

```bash
bash scripts/build-site.sh
```

Then open:
- `dist/index.html`
- `dist/quiz/<slug>/index.html`
- `dist/discussion/<slug>/index.html`

## Legacy imports

Legacy PPTX material is treated as raw source, not canonical repo content.

Current imported references:
- `legacy-import/discussion/`
- `legacy-import/quiz/external_bundle_reference.md`

Use those files as staging inputs when normalizing old decks into Marp Markdown.

## GitHub Pages

Required setting:
- **Settings -> Pages -> Build and deployment -> Source: GitHub Actions**

The workflow deploys on:
- push to `main`
- manual dispatch

## Troubleshooting

If the site shows 404 or old content:
- confirm the latest `static.yml` run succeeded
- inspect the failed run log before changing Pages settings
- verify `dist/index.html` and deck outputs are being generated by `bash scripts/build-site.sh`
- hard refresh the published URL after a successful deploy
